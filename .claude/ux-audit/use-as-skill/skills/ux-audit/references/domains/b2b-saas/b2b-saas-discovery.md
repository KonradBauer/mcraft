## SaaS Discovery Guidance

### Key Assumptions to Form
- **ACV range**: <$1K (self-serve) / $1K–$25K (hybrid) / >$25K (enterprise)?
- **Sales motion detail**: PLG / sales-led / hybrid? Free trial vs. freemium vs. demo-only?
- **Developer audience**: Is there a developer adoption path preceding enterprise purchase?
- **Partner/channel model**: Direct only / partner-assisted / partner-primary?
- **Demo/trial flow**: What is the critical conversion path? (self-serve signup / guided demo / sandbox)
- **Pricing transparency**: Expected in PLG; optional in enterprise

### Common Project Shapes
- **SaaS startup**: One-pager or landing + subpages; Product-Led archetype
- **Growth-stage SaaS**: Multi-page; Solution-Led archetype
- **Enterprise SaaS**: Multi-page; Solution-Led or Authority-Led archetype
- **Developer tools**: Multi-page; Developer / Community-Led archetype
- **Channel-heavy**: Multi-page + partner sub-site; Partner / Channel-Led archetype
- **Two-sided platform**: Multi-page with dual-audience routing; Marketplace archetype

### Unique Constraints
- **Integration pages**: Key for platform products — answers "Does it work with my stack?"
- **Demo/trial flow**: Critical conversion path — must be frictionless
- **Documentation hub**: For developer-facing products, docs may be the most-visited section
- **Partner sub-site**: Channel-led products need a self-contained partner section with its own hero, benefits, and application CTA
- **Dual-audience routing**: Marketplace sites need explicit audience routing without cannibalizing either side's messaging
