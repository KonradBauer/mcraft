# Service-Utility: Archetypes

Domain knowledge for service-utility site archetypes and their strategic trade-offs.
Part of: references/domains/service-utility/

---

## Archetype 1: Free Tool with Monetization

**Optimizes for:** tool usage leading to premium upsell or ad revenue.

**Best fit:** online calculators (mortgage, BMI, currency), converters (file format, unit), generators (password, color palette, lorem ipsum).

**Sacrifices:** brand storytelling — the tool IS the experience. Homepage = the tool interface.

**Examples:** Canva (free design tool → premium), TinyPNG (free compression → API).

**Structural signature:**

- Landing page IS the tool — no marketing homepage
- Value delivered before any ask (signup, payment, ad interaction)
- Premium features visible but not blocking core functionality
- SEO-optimized tool pages as primary acquisition channel

**Revenue mechanics:**

- Freemium: core tool free, advanced features gated (export options, batch processing, API access)
- Ad-supported: display ads around tool interface (careful not to interfere with usability)
- Usage limits: free tier with daily/monthly caps, paid tier unlimited

**Key UX tension:** monetization elements must never degrade the core tool experience. The moment the free tool feels crippled, trust collapses.

## Archetype 2: Comparison/Aggregator Service

**Optimizes for:** affiliate conversion through comparison.

**Best fit:** price comparison (energy, insurance, flights), review aggregators, "best X for Y" tools.

**Structure:** input criteria → results with affiliate links.

**Examples:** Kayak (flight comparison), Uswitch (energy comparison), NerdWallet (financial product comparison).

**Structural signature:**

- Input form as hero (criteria collection)
- Results page with ranked/sortable options
- Each result links out to provider (affiliate model)
- Trust signals critical: "we compare X providers," update timestamps, methodology disclosure

**Revenue mechanics:**

- Affiliate commissions per click-through or conversion
- Sponsored/featured placement (must be disclosed)
- Lead generation: collecting user details to pass to providers

## Archetype 3: Professional Service Tool

**Optimizes for:** demonstrating expertise to convert to paid service.

**Best fit:** SEO audit tools (Moz, Ahrefs free tools), legal document generators, financial planning calculators.

**The free tool is a lead magnet for professional services.**

**Examples:** Moz (free SEO tools → paid suite), HubSpot Website Grader (free audit → CRM platform), Termly (free privacy policy generator → compliance service).

**Structural signature:**

- Tool delivers genuine, useful result (not a teaser)
- Result includes insights the user cannot fully act on alone
- CTA positioned as "we can help you fix/improve this"
- Professional credibility signals throughout (certifications, client logos, expertise markers)

**Revenue mechanics:**

- Lead qualification through tool usage (tool inputs reveal user's situation)
- Consultation booking as primary CTA
- Email capture via "send full report" or "monitor changes"
- Tool result as conversation starter for sales team

**Key UX tension:** the free result must be genuinely valuable, not artificially limited. Users who feel tricked by a shallow result will never convert to paid services. The tool proves competence; the service sells itself through demonstrated expertise.
