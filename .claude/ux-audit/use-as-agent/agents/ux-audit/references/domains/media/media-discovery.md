Domain knowledge for media / content publication sites.

## Media Discovery Guidance

### Key Assumptions to Form
- **Revenue model**: Advertising / subscriptions / sponsorship / affiliate / newsletter with paid tier?
- **Publishing frequency**: Daily / 3-5x per week / weekly / monthly?
- **Content creators**: Full editorial team / single author / contributors?
- **Primary traffic source**: Search (SEO-optimized) / social (share-optimized) / email (newsletter-first) / direct?
- **Newsletter role**: Is the newsletter the product or a retention tool?
- **Content format**: Written articles / video / podcast / mixed?
- **Sponsored content**: Does the site run native advertising or sponsored posts?

### Common Project Shapes
- **Online magazine**: Content-First archetype, daily publishing, ad-supported, multiple categories
- **Independent newsletter**: Newsletter-First archetype, single author, paid tier, minimal website
- **Niche publication**: Content-First archetype, weekly, community element, subscription
- **Video-first media**: Multimedia archetype, video player UX, show/series structure

### Unique Constraints
- **Article page > Homepage**: Most traffic lands on articles from search/social — article page design matters more than homepage
- **Ad integration**: Must not destroy reading experience (no interstitial mid-paragraph, no layout shifts)
- **Paywall strategy**: Hard paywall (subscription only) vs. metered (X free articles) vs. freemium (free + premium)
- **SEO**: Structured data (Article schema), publication date, author markup, canonical URLs
- **Content freshness**: Homepage must feel alive — stale content signals abandoned publication
- **RSS / API**: Power readers expect RSS feeds; content aggregators need structured data

### Default Assumptions
If no info provided: Blog/publication with 2-3 content pillars, publishing 3-5x per week, monetized through display ads + newsletter, 60% mobile traffic, 50% from search, newsletter is primary conversion goal.
