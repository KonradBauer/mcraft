# Modular Section Patterns

Assemble these atomic blocks to form a page strategy.

## 0. Navigation Aids (Wayfinding)
### A. Breadcrumbs
- **Rule**: Mandatory for pages > 2 levels deep.
- *Examples*: `Home > Solutions > Healthcare > Case Study`
- *Purpose*: Orientation and "Escape Hatch" (Front Door Principle).

### B. Contextual Cross-Links
- **Rule**: Place "Next Step" or "Related Info" links in the content flow.
- *Examples*:
    - On Product Page: "Compare with X", "View Integration Docs"
    - On Support Page: "Contact Sales"
    - On Blog Post: "View related Product" (Sidebar)

## 1. Hero Sections
### A. The "Promise" Hero (Context-First)
- **H1**: [Capability Promise] - What can you DO?
- **Sub**: [Method] - How does it work?
- **Visual**: UI Screenshot or Context Shot.
- **CTA**: High friction (Demo) or Low friction (Try).

### B. The "Trust" Hero (Service/Expert)
- **H1**: [Value Proposition] + [Human Element].
- **Sub**: [Empathy Statement] - "We understand..."
- **Visual**: Photo of the Expert/Team.
- **CTA**: "Book Consultation".

## 2. Trust Anchors
### A. The Logo Bar
- Use immediately below Hero. "Trusted by [X, Y, Z]".
### B. The Stat Bar (Service)
- "10 Years", "$50M Generated", "500+ Clients".

## 3. The "Pain" Block (Agitation)
- **Header**: "Stop wasting time on X".
- **Visual**: "Old Way" (Spreadsheet/Chaos).
- **Copy**: Describe the struggle.

## 4. The "Zig-Zag" (How it Works)
- Alternating Layout (Text Left/Image Right -> Image Left/Text Right).
- **Block 1**: Feature A -> Benefit A.
- **Block 2**: Feature B -> Benefit B.
- **Block 3**: Feature C -> Benefit C.

## 5. The "Switch" (Objection Handling)
- **Header**: "Switching is easy".
- **Content**: Onboarding timeline, Concierge migration.

## 6. Social Proof (Deep)
- Testimonials, Case Study Cards, Video reviews.

## 7. Case Study Block (Atomic Deployment)

Reusable "atoms" to distribute across the site.

### A. The Case Study Card
```
┌─────────────────────────────────┐
│ [LOGO]  Industry: Healthcare    │
│ ─────────────────────────────── │
│ "Quote from client..."          │
│                                 │
│ +300% ROI │ 2-Week Setup        │
│ [Read Case Study →]             │
└─────────────────────────────────┘
```

### B. The "Glanceable" Header (for full case study page)
```
+-------------+-------------+-------------+
|   300% ROI  |  2-Week     |  $2M        |
|             |  Setup      |  Saved      |
+-------------+-------------+-------------+
```

### C. Contextual Quote (for Pricing/Checkout)
- Single testimonial placed next to friction points.
- Example: *"Setup took 20 minutes"* – near Demo button.

## 8. Full Page Pattern: The Case Study (QuadGraphics Style)

Structure for a dedicated Case Study page.

1. **Header**: Client Name + Core Result (The "Glanceable" Header).
2. **Challenge** (The Problem):
    - What was the situation before?
    - What were the pain points?
3. **Solution** (The Methodology):
    - How did we solve it?
    - Specific features/services used.
4. **Results** (The Outcome):
    - Bullet points with specific metrics.
    - "Before vs After" comparison.
5. **Testimonial**: The "Hero Quote" from the decision maker.
6. **CTA**: "Get similar results".

---

## 9. The Close (Final CTA)

A standalone section with strong command to act.

- **Header**: Repeat core value prop (e.g., "Ready to automate?").
- **Risk Reversal**: "No credit card required", "Cancel anytime", "30-day guarantee".
- **CTA**: Large, visually distinct button.

---

## 10. The Ultimate Guarantee (Risk Reversal)

Go beyond "30-day refund".

- **Name It**: Give it a branded name (e.g., "The Iron-Clad 100% Satisfaction Promise").
- **Balance the Risk**: State clearly: *"We take all the risk. You take none."*
- **Positive Language**: *"We guarantee you will love this"* (not *"If you don't like it"*).
- **The "Unlikely Event"**: *"In the unlikely event you don't see results, we will refund you + pay for your coffee."*

## 11. The Objection Crusher (Strategic FAQ)

Don't just answer random questions. Address the 4 core barriers to sale.

1.  **"I don't understand what this is."** -> (Clarify the Offer)
2.  **"I don't know why I need it."** -> (Reiterate Benefits/Pain)
3.  **"I don't believe it will work."** -> (Link to Case Study/Proof)
4.  **"It's too expensive."** -> (Reframing: ROI vs Cost of Inaction)
